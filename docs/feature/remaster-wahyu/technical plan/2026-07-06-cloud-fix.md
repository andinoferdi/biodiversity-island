# Cloud Toggle Natural Fade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Do not use subagents for this small follow-up.

**Goal:** Make Clouds On and Clouds Off transition naturally, matching the feel of Rain On and Rain Off, instead of appearing or disappearing instantly.

**Architecture:** Keep `Clouds` mounted at all times, pass the cloud toggle as a target blend, and let `Clouds.tsx` animate visibility, altitude, shadow opacity, and scale through refs in `useFrame`. This mirrors `Rain.tsx`, whose natural off animation works because the component remains mounted while `useEasedBlend(false)` eases down to zero.

**Tech Stack:** Next.js 16, React 19, TypeScript, Three.js 0.185, @react-three/fiber v9, Tailwind CSS v4, npm.

---

## Root Cause

`Rain` feels natural because `IslandScene.tsx` always renders:

```tsx
<Rain
  key={graphicQuality}
  isRaining={isRaining}
  timeScale={timeScale}
  graphicQuality={graphicQuality}
/>
```

Inside `Rain.tsx`, `useEasedBlend(isRaining)` keeps moving the blend toward `0` after Rain Off, so drops fade out before becoming invisible.

`Clouds` does not get that chance. `IslandScene.tsx` currently renders:

```tsx
{isCloudy && <Clouds isRaining={isRaining} timeScale={timeScale} />}
```

When Clouds Off is clicked, React unmounts the whole component immediately. `Clouds.tsx` only receives `isRaining`, not the cloud toggle itself, so it can animate rain-darkening but cannot animate cloud presence.

The fix should not redesign the cloud art. It should preserve the existing deterministic low-poly cloud layout, wind drift, rain darkening, and shadow decals, while adding one new eased cloud-presence blend.

## Task 1: Keep Clouds Mounted and Add Presence Blend

**Files:**
- Modify: `src/components/prototype/effects/Clouds.tsx`
- Modify: `src/components/prototype/IslandScene.tsx`

**Interface:**
- Input: `isCloudy: boolean`, `isRaining: boolean`, `timeScale: TimeScale`.
- Output: `Clouds` remains mounted and animates toward visible or hidden state over the same eased duration used by rain.

- [ ] **Step 1: Update `CloudsProps`**

In `src/components/prototype/effects/Clouds.tsx`, change the props interface from:

```ts
interface CloudsProps {
  isRaining: boolean;
  timeScale: TimeScale;
}
```

to:

```ts
interface CloudsProps {
  isCloudy: boolean;
  isRaining: boolean;
  timeScale: TimeScale;
}
```

Then change the component signature from:

```ts
export default function Clouds({ isRaining, timeScale }: CloudsProps) {
```

to:

```ts
export default function Clouds({ isCloudy, isRaining, timeScale }: CloudsProps) {
```

- [ ] **Step 2: Add cloud presence blend**

Inside `Clouds`, keep the existing rain blend:

```ts
const rainBlend = useEasedBlend(isRaining);
```

Add a second blend directly after it:

```ts
const cloudBlend = useEasedBlend(isCloudy);
```

This must stay separate from `rainBlend`. `rainBlend` controls storm color and rain sink. `cloudBlend` controls whether the cloud layer itself is present.

- [ ] **Step 3: Add named constants for hide/show motion**

Near the existing constants in `Clouds.tsx`, add:

```ts
const CLOUD_FADE_MIN_SCALE = 0.55;
const CLOUD_HIDE_LIFT = 1.2;
const CLOUD_VISIBLE_OPACITY = 0.92;
const SHADOW_VISIBLE_OPACITY = 0.34;
```

Use these meanings:
- `CLOUD_FADE_MIN_SCALE`: hidden clouds shrink but do not collapse to zero, avoiding a harsh pop.
- `CLOUD_HIDE_LIFT`: Clouds Off lifts clouds slightly while they fade, like they are thinning into the sky.
- `CLOUD_VISIBLE_OPACITY`: existing cloud opacity value.
- `SHADOW_VISIBLE_OPACITY`: maximum shadow decal opacity; this is intentionally softer than the texture's internal alpha.

- [ ] **Step 4: Animate cloud material opacity**

In `useFrame`, after reading both blends:

```ts
const storm = rainBlend.current;
const presence = cloudBlend.current;
```

Replace the existing:

```ts
const blend = rainBlend.current;
```

and replace later references to rain color/sink with `storm`.

Then update the cloud material each frame:

```ts
const mat = lobeMesh.material as THREE.MeshLambertMaterial;
mat.color.lerpColors(colors.clear, colors.rain, storm);
mat.opacity = CLOUD_VISIBLE_OPACITY * presence;
lobeMesh.visible = presence > 0.01;
```

Do not return early before updating matrices while `presence > 0.01`, because fade-out still needs frames.

- [ ] **Step 5: Animate lobe scale and altitude**

In the lobe loop, replace:

```ts
const y = spec.y - blend * RAIN_SINK;
```

with:

```ts
const visibleScale = CLOUD_FADE_MIN_SCALE + (1 - CLOUD_FADE_MIN_SCALE) * presence;
const y = spec.y - storm * RAIN_SINK + (1 - presence) * CLOUD_HIDE_LIFT;
```

Then replace:

```ts
dummy.scale.set(lobe.scale, lobe.scale * 0.72, lobe.scale);
```

with:

```ts
dummy.scale.set(
  lobe.scale * visibleScale,
  lobe.scale * 0.72 * visibleScale,
  lobe.scale * visibleScale,
);
```

This makes Clouds Off feel like clouds thinning and lifting, rather than teleporting away.

- [ ] **Step 6: Animate shadow opacity and visibility**

In the material setup for the shadow instanced mesh, replace:

```tsx
<meshBasicMaterial map={shadowTexture} transparent depthWrite={false} />
```

with:

```tsx
<meshBasicMaterial
  map={shadowTexture}
  transparent
  opacity={SHADOW_VISIBLE_OPACITY}
  depthWrite={false}
/>
```

In `useFrame`, after the cloud material update, add:

```ts
if (shadowMesh) {
  const shadowMat = shadowMesh.material as THREE.MeshBasicMaterial;
  shadowMat.opacity = SHADOW_VISIBLE_OPACITY * presence;
  shadowMesh.visible = presence > 0.01;
}
```

Keep the existing shadow matrix updates so shadows continue following clouds during fade in/out.

- [ ] **Step 7: Keep drift natural during toggle**

Keep the existing movement:

```ts
const move = delta * timeScale;
center.x += WIND.x * WIND.speed * spec.driftScale * move;
center.z += WIND.z * WIND.speed * spec.driftScale * move;
```

Do not reset `centers.current` when `isCloudy` changes. Clouds On should reappear from wherever the cloud layer drifted to, not jump back to initial positions.

If visual verification shows the hidden cloud layer drifting too far while Off for a long time, do not reset it in this task. The current wrap logic already keeps it bounded.

- [ ] **Step 8: Always render `Clouds` from `IslandScene`**

In `src/components/prototype/IslandScene.tsx`, replace:

```tsx
{isCloudy && <Clouds isRaining={isRaining} timeScale={timeScale} />}
```

with:

```tsx
<Clouds isCloudy={isCloudy} isRaining={isRaining} timeScale={timeScale} />
```

This is the core behavioral fix. Without this, Clouds Off will always remain instant because React removes the component.

## Task 2: Verify Cloud Toggle Motion Against Rain Toggle

**Files:**
- No source edits unless verification exposes a bug in Task 1.

**Interface:**
- Input: local dev server and browser.
- Output: manual evidence that Clouds On/Off fades naturally and does not regress Rain On/Off.

- [ ] **Step 1: Start dev server**

Run:

```bash
rtk npm run dev
```

Open the printed local URL, usually:

```text
http://localhost:3000
```

If port 3000 is occupied by a user-owned server, do not kill it. Use the available port printed by Next.

- [ ] **Step 2: Verify Clouds Off**

In the browser, start with Clouds On. Click `Clouds Off`.

Expected:
- Clouds fade over about 2.5 seconds.
- Clouds lift slightly while fading.
- Cloud shadows fade with the clouds.
- No cloud disappears in a single frame.
- The terrain and animals remain visible and unaffected.

- [ ] **Step 3: Verify Clouds On**

Click `Clouds On`.

Expected:
- Clouds fade in over about 2.5 seconds.
- Clouds settle down from the lifted hidden position.
- Cloud shadows fade in with the clouds.
- Clouds continue drifting in the same wind direction.
- No cloud snaps back to the original seed position.

- [ ] **Step 4: Verify Rain and Cloud interactions**

Run these combinations:

1. Clouds On, Rain Off.
2. Clouds On, click Rain On.
3. Rain On, click Clouds Off.
4. Rain On, click Clouds On.
5. Clouds On, click Rain Off.

Expected:
- Rain On still darkens clouds and lowers them gradually.
- Rain Off still brightens clouds gradually.
- Clouds Off while raining fades the cloud layer out; rain streaks continue.
- Clouds On while raining fades in gray/rain-state clouds, not bright clear clouds.
- No one-frame color pop occurs when both blends are active.

- [ ] **Step 5: Verify Pause behavior**

Click `Pause`, then toggle Clouds Off and Clouds On.

Expected:
- The transition blend should still animate because `useEasedBlend` uses raw frame `delta`, matching rain/sky behavior.
- Cloud drift should remain paused because drift multiplies by `timeScale`.
- This matches the current rain design: weather transition can animate while simulation movement is paused.

- [ ] **Step 6: Verify quality tiers**

Switch Low, Medium, and High quality while Clouds On and Clouds Off.

Expected:
- Cloud fade still works in every quality tier.
- Low quality still disables the existing low-quality features controlled elsewhere.
- No console error appears when quality changes.
